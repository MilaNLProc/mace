How it works

Simply upload an Excel file, and let us do the rest.

Let's explain MACE with an example. When evaluating redundant annotations (like those from Amazon's Mechanical Turk), we usually want to aggregate annotations to

  • recover the most likely answer
  • find out which annotators are trustworthy
  • evaluate items and task difficulty

MACE solves all of these problems, by learning competence estimates for each annotators and computing the most likely answer based on those competences. Furthermore, if you have a ground truth (here called "control"), it can be used as a feature to help the algorithm.

To use MACE you need

Annotations

An Excel file where each row corresponds to an instance that has to be annotated and each column correspond to one annotator. The annotator columns can be named freely (e.g., using the annorators names or simply using an annotator id).

Controls (optional)

An optional column named CONTROLS, in the same Excel file, which corresponds to the ground truth; this column will help the algorithm in producing the competence estimates and, ultimately, produce the most probable annotattion for each instance.

NOTE: The only restriction is that apart from the annotators columns and an optional CONTROLS column, the file should not include any other column, otherwise it will be considered an annotation.

Steps

Let's get practical

1. Prepare your data: Add annotations

Create an Excel file with two columns where each row corresponds to an instance that has to be annotated and each column correspond to one annotator. Each annotator should have its own column, even if it did not annotate some of the texts (in such cases just leave the cell empty). You can name the columns as you prefer, for example using the annotators' names or simply using an annotator id

2. Add CONTROLS (optional)

If for some instances you know the correct annotation, you can provide it in a separate column named CONTROLS (uppercase); this column will help the algorithm producing the competence estimates and, ultimately, produce the most probable annotattion for each instance.

3. Advanced Settings (optional)

If for each possible annotation you have a prior belief, you can provide it in a separate Excel file through the Advanced Settings menu. This Excel file must have only two columns: one with the possible unique annotations, named arbitrarily (e.g., Annotations), and one with the prior probabilities named PRIORS (uppercase). This prior information will inform the algorithm producing the most probable annotation for each instance. If the number of labels for which you specify a prior is smaller than the number of unique labels in the annotation the prior for the missing lable will be set to zero and you will be notified. In addition, you can further customize the algorithm twicking the available parameters in the Advanced Settings menu.

4. Receive the output

Once the file is ready, upload the file. Depending on the number of requests, it can take up to 30 minutes (but usually) 3-4 are enough). No data is stored on our server. Once we receive your file, we will process it and send you the output via email. The output file is an Excel file composed by 3 worksheets:

F.A.Q

Frequently Asked Questions

  • What is MACE?

    In settings where you have multiple annotations of a text, MACE is way to recover the most likely answer, find out which annotators are trustworthy, evaluate items and task difficulty.

  • What happens to my data?

    Nothing. We never store the data you upload on disk: it is only kept in memory for the duration of the modeling, and then deleted. We do not retain any copies or traces of your data.

  • What input formats do you support?

    All uploaded files must have the .xlsx format.

  • How does it work?

    It uses a variant of Stability Selection (Meinshausen and Bühlmann, 2010) to fit hundreds of logistic regression models on random subsets of the data, using different L1 penalties to drive as many of the term coefficients to 0. Any terms that receive a non-zero coefficient in at least 30% of all model runs can be seen as stable indicators.
